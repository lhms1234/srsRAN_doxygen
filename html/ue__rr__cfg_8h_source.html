22 /*----------------------------------------------------------------------------------------------------------------------
23  * \file
24  * These .h/.cc files contain the functions responsible for setting the ASN1 fields
25  * related to radio resource configuration from the eNB side (see TS 36.331 - 5.3.10)
26  *
27  * The following function naming convention is followed:
28  * - suffix "_enb_cfg" - any function related to setting asn1 fields based on the configuration passed to the eNB
29  * through parsed configuration files
30  * - suffix "_setup" - the asn1 fields are set using the respective "_enb_cfg" function as default, plus any other
31  * parameter known at runtime (e.g. SR/CQI resources). Functions with this suffix are called
32  * during the rrcConnectionSetup and rrcConnectionReestablishment proceduress
33  * - suffix "_reconf" - sets the asn1 fields that are known during the rrcConnectionReconfiguration. That includes
34  * "_enb_cfg" + "_setup" fields + DRBs + SCells, etc. It compares the current UE RRC configuration
35  * (accumulation of the RRC messages sent so far), with the next one that will result from
36  * the RRC setup/reestablishment/reconfiguration procedure. Based on this comparison, it will
37  * fill the output ASN1 struct with fields that are strictly necessary
38  * - suffix "_diff" - update the struct relative to the current UE RRC config with the fields present in
39  * the Reconf/Setup/Reestablishment message that is about to be sent to the UE
40  *
41  *--------------------------------------------------------------------------------------------------------------------*/
